# Peter — 2026-04-14 — C10 through C13

I went through 4 more cases so far comparing extraction quality and
downstream reasoning. A few consistent patterns came up.

## Main findings from C10–C13

1. **The engine often confuses "missing required item" with "alternative
   pathway not present."** This was the biggest recurring issue in the
   analysis layer. The app lists many absent criteria as "gaps" even when
   they are just alternative PET justification pathways that do not apply
   to that case. Fix: separate true blocking deficiencies from optional
   alternative pathways not present.

2. **Case-level reasoning is usually directionally right, but the narrative
   sometimes overreaches beyond the taxonomy.** Especially in C11 and C12,
   the appeal narrative tried to rescue the case using logic that was
   clinically arguable but not actually supported by the coded taxonomy.
   Example: using non-qualifying ECG abnormalities as if they were formal
   PET-supporting ECG criteria. Fix: force narrative to stay inside the
   scored criterion framework unless explicitly labeled "supplemental
   clinical argument."

3. **Extraction still mixes data types across fields.** ECG findings
   promoted to comorbidities; lab abnormalities promoted to diagnoses;
   family history and smoking history under comorbidities; prior tests
   duplicated in imaging and procedures; resolved symptoms kept as active.
   Fix: tighten schema separation (diagnosis / symptom / test finding /
   prior study / risk factor / family history / resolved history).

4. **The app needs stronger contradiction handling.** "Dizziness" or
   "syncope" appearing under old problem lists while ROS says not present.
   Fix: recency/context weighting — current HPI/ROS/assessment should
   outrank old problem list; resolved items should not remain active unless
   reaffirmed.

5. **Some extraction errors are small, but some are dangerous.** In C12
   (second extraction), unsupported labs with future dates were inserted.
   Fix: hard rule — never emit a structured lab unless directly anchored
   to source text.

## Case-specific takeaways

- **C10** — Extraction usable; analysis directionally correct; biggest issue
  was too many "gaps" that were actually non-applicable alternate pathways.
  MED logic also missed that at least one med had a start date.
- **C11** — True weakness (inability to exercise not explicitly stated)
  correctly caught. Narrative overreached using RBBB/LAFB as if formal
  PET pathways.
- **C12** — Engine correctly concluded not a good PET case. Main problem
  was explanation style: too many noisy gaps instead of simply saying
  there's no valid PET pathway. Comparing extraction modes, one
  extraction was cleaner structurally but introduced unsupported labs.
- **C13** — Best case so far. EX-001 correctly identified (inability to
  do TST due to dyspnea/edema with large breasts noted). Remaining issue
  was presentation (too many non-applicable pathways shown as gaps) plus
  mild over-inference (hypothyroidism from levothyroxine).

## Claude Only vs Comprehend + Claude

- **Claude Only** — sometimes messier structurally, but stays closer to
  source; fewer dangerous unsupported insertions.
- **Comprehend + Claude** — sometimes cleaner-looking organization but more
  prone to wrong field placement and, in C12, unsupported data in
  structured output.

Impression: Claude Only is safer. Comprehend + Claude may help extraction
formatting but needs stronger guardrails before I'd trust it more.

## Most useful fixes to prioritize next

1. Distinguish blocking deficiencies from non-applicable alternatives
2. Constrain narrative generation to the scored taxonomy
3. Tighten field typing
4. Add contradiction-resolution and recency weighting
5. Strict source-anchoring for labs and structured values
6. Reduce final output to the top 1–3 real reasons
