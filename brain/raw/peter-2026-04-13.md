Main findings from the first pass:

1. **MED-002 (C1)** — Meds listed, but incomplete start dates and no clear
   ≥6-week duration. True value: not met (2/1). System: met (1/1).
   → Presence of meds is being treated as sufficient, without enforcing
   duration/completeness.

2. **SX-001 (C2, C4, C5)** —
   - C2: single fleeting episode
   - C4: dyspnea on exertion, no baseline comparison
   - C5: explicitly "no new complaints"
   True value: not met. System: met.
   → Symptoms detected, but no verification of change vs baseline.

3. **SX-002 (C5)** — Symptoms present, but no onset, frequency, or
   progression documented. True value: not met. System: met.
   → Symptom presence is being treated as a complete timeline.

4. **EX-001 (C4)** — Dyspnea on exertion present, but no explicit inability
   to exercise. True value: not met. System: met.
   → Dyspnea interpreted as functional limitation without explicit causal
   statement.

**Pattern:** the system maps presence of a related feature → "met", without
verifying that the full definition is satisfied.

**Underlying theme:** the remaining gap is at the final classification step,
specifically enforcing:
- constraints (e.g., duration, required fields)
- temporal logic (change vs baseline)
- structured completeness (timeline elements)
- causal linkage (explicit reason → conclusion)

**Summary:** consistently requiring that all components of a criterion
definition are explicitly satisfied before assigning "met."
