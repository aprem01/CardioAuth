# Validation batch C1–C5

**Date:** 2026-04-13 · **Reviewer:** [Peter](../people/peter.md)

## Cases

- **C1** — partial — MED-002 marked met despite missing duration; drove the start_date_or_duration required_element
- **C2** — partial — SX-001 marked met on single fleeting episode; drove change_vs_baseline required_element
- **C3** — pass-directional — analysis generally correct
- **C4** — partial — EX-001 marked met on "dyspnea on exertion" with no explicit causal link to inability to exercise
- **C5** — partial — SX-001 marked met despite explicit "no new complaints"; SX-002 marked met without onset or frequency

## Patterns observed

Uniform pattern across cases: **presence of a feature being treated as satisfaction of the criterion definition.** The reasoner saw the keyword (dyspnea, meds listed, symptoms present) and concluded the criterion was met, regardless of whether the full conjunctive definition was satisfied.

Also: EX-001 was structurally missing from CPT 78492's applicable set — it had only been tagged for SPECT codes. This was a separate taxonomy bug caught during the same batch.

## Fixes shipped in response

- `a0d4a07` — EX-001 `applies_to` fix + regression test
- `ab32522` — `required_elements` + structural enforcement for MED-002, SX-001, SX-002, EX-001, MED-001
- Taxonomy expansion: all 31 criteria now have required_elements (100% coverage)

## Remaining gaps

None from this batch directly — surfaced issues were all addressed. See [validation/c10-c13.md](c10-c13.md) for next-batch findings.
