# Validation batch C10–C13

**Date:** 2026-04-14 · **Reviewer:** [Peter](../people/peter.md)

## Cases

- **C10** — extraction usable; analysis directionally correct; biggest issue was too many non-applicable pathways shown as "gaps"
- **C11** — correctly identified weak documentation (inability to exercise not stated). Narrative overreached using RBBB/LAFB as if they were formal PET-supporting ECG pathways.
- **C12** — engine correctly concluded this was not a valid PET case. BUT Comprehend+Claude extraction inserted unsupported labs with future dates. SAFETY issue, not cosmetic.
- **C13** — best case so far. EX-001 correctly identified (inability to do TST due to dyspnea/edema). Mild over-inference: hypothyroidism inferred from levothyroxine.

## Patterns observed

1. **Blocking vs alternative conflation** — unmet alternative pathways were flagged as gaps even when another pathway member was met. Biggest noise source.
2. **Narrative over-reach** — appeal narrative used clinical logic outside the coded taxonomy to rescue cases.
3. **Schema bleed** — ECG → comorbidities, family history → comorbidities, resolved symptoms → active problems, prior tests duplicated across imaging and procedures.
4. **Contradiction handling missing** — old problem list contradicted by current ROS; system wasn't resolving in favor of current.
5. **Dangerous extraction** — unsupported labs with future dates (C12 specifically). Safety-critical.
6. **Medication → diagnosis over-inference** — levothyroxine → hypothyroidism (C13).

## Claude-Only vs Comprehend+Claude

Peter's assessment:

- **Claude Only** — sometimes messier structurally, but stays closer to source, fewer dangerous insertions. **Safer default.**
- **Comprehend+Claude** — sometimes cleaner-looking organization, but more prone to wrong field placement AND unsupported-data insertion (the C12 future-date labs came from this path). Better at entity pickup; worse at preserving relationships.

Conclusion: Claude Only is the safer default; Comprehend+Claude needs stronger guardrails before being trusted more.

## Fixes shipped in response

- `982af0d` — core six:
  - `pathway_group` on Criterion + `classify_gaps()` splitting blocking / alternative_not_used / supporting_unmet
  - Narrative stay-in-taxonomy rule + `supplemental_clinical_argument` field
  - ChartData v2 (see [decisions/2026-04-14-chartdata-v2.md](../decisions/2026-04-14-chartdata-v2.md))
  - Contradiction + recency precedence (ROS > HPI > PMH > old problem list)
  - `LabResult.source_anchor` + `validate_lab_source_anchoring()` drops future-dated labs
  - `headline_summary` field (top 1-3 reasons)

- `faad27c` — three follow-ups:
  - Lab drop warnings now surface in `system_warnings` (physician-visible)
  - Explicit "don't infer diagnosis from medication" negative example (levothyroxine/hypothyroidism named)
  - Extraction mode badge on review screen (🛡 Claude Only / ⚠ Comprehend + Claude)

## Remaining gaps

- Peter's 20-30 retrospective labeled cases not yet requested — needed for first real sensitivity/specificity numbers.
- Comprehend+Claude needs additional guardrails before it should be recommended — today it's visibly marked as higher-risk, not strictly safer.
